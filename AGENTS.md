# AGENTS.md

## Project Rules
- Перед началом значимых изменений проверяй `SPEC.md`.
- Если изменилась структура проекта (новые страницы, ключи `localStorage`, схема данных, flow синхронизации правил, новые скрипты), обязательно обновляй `SPEC.md` в этом же наборе изменений.
- Поддерживай проект как статический (`HTML + CSS + JS`), без добавления сборщиков по умолчанию.
- Не удаляй пользовательские данные из `localStorage` миграциями без явной причины.

## Rules Snapshot Workflow
- Источник правил: `https://pf2.ru/rules`.
- Обновление локального снапшота: `scripts/download_pf2_rules.sh`.
- Перегенерация встроенных ancestry/feats: `node scripts/generate_pf2_defaults.js`.
- После изменений сверяй, что `data/pf2-defaults.js`, `README.md` и `SPEC.md` согласованы.

## Quality Gate
Перед завершением изменений:
1. Проверь, что страницы открываются без ошибок JS.
2. Проверь CRUD ancestry и ancestry feats.
3. Проверь выбор ancestry feats в `character.html` и лимиты по уровням 1/5/9/13/17.
4. Проверь, что ссылки на снапшот правил (`rules.html`) валидны.
