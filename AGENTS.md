# AGENTS.md

## Project Rules
- Перед началом значимых изменений проверяй `SPEC.md`.
- Если изменилась структура проекта (новые страницы, ключи `localStorage`, схема данных, flow интеграции документации), обязательно обновляй `SPEC.md` в этом же наборе изменений.
- Поддерживай проект как статический (`HTML + CSS + JS`), без добавления сборщиков по умолчанию.
- Не удаляй пользовательские данные из `localStorage` миграциями без явной причины.

## External Docs Workflow
- Источник внешней документации может меняться; не хардкодь домен в коде и документации без явного запроса.
- После добавления/замены локального архива правил сверяй, что `rules.html`, `README.md` и `SPEC.md` согласованы.

## Quality Gate
Перед завершением изменений:
1. Проверь, что страницы открываются без ошибок JS.
2. Проверь CRUD ancestry и ancestry feats.
3. Проверь выбор ancestry feats в `character.html` и лимиты по уровням 1/5/9/13/17.
4. Проверь, что ссылки на локальную документацию правил (`rules.html`) валидны.
